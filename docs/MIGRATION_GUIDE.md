# Migration Guide: video_streaming → Template Pattern

The `video_streaming` project predates framework templates. Its modules use raw manifests (~190 lines each). The `erp_back` project demonstrates the recommended template pattern (~50 lines each, 74% reduction).

---

## Why Migrate?

| Aspect | Raw Pattern (video_streaming) | Template Pattern (erp_back) |
|---|---|---|
| **Lines per module** | ~190 lines | ~50 lines |
| **Boilerplate** | Full K8s manifest definitions | Auto-generated by templates |
| **Maintenance** | Edit every manifest field manually | Change only config values |
| **Validation** | No compile-time checks | `check` blocks in builders |
| **Consistency** | Manual label/selector matching | Auto-generated from `name` |
| **New features** | Must add to each module | Inherit from updated template |

---

## Migration Steps Per Module

### Step 1: Identify the Module Type

| Raw Module Pattern | Target Template |
|---|---|
| Deployment + Service + ConfigMap | `WebAppModule` |
| StatefulSet + PV + PVC | `SingleDatabaseModule` |
| Kafka CRDs | `KafkaClusterModule` |
| PostgreSQL CRDs | `PostgreSQLClusterModule` |
| MongoDB CRDs | `MongoDBCommunityModule` |
| RabbitMQ CRDs | `RabbitMQClusterModule` |
| Redis CRDs | `RedisModule` |

### Step 2: Create New Module Definition

```kcl
# BEFORE (raw pattern — ~190 lines):
import framework.models.modules.component as component
import framework.builders.deployment as deploy
import framework.builders.service as svc
import framework.builders.configmap as cm
import framework.builders.leader as leader
import framework.builders.service_account as sa

schema MyAppModule(component.Component):
    kind = "APPLICATION"
    _deployment = deploy.build_deployment(deploy.DeploymentSpec {
        name = name
        namespace = namespace
        image = image
        version = version
        port = 8080
        replicas = 2
        env = [
            { name = "SPRING_PROFILE", value = "dev" }
            { name = "DB_HOST", value = "postgres.data.svc" }
        ]
        resources = deploy.ResourceSpec {
            cpuRequest = "250m"
            memoryRequest = "512Mi"
            cpuLimit = "1"
            memoryLimit = "2Gi"
        }
        livenessProbe = deploy.ProbeSpec {
            probeType = "http"
            path = "/health"
            port = 8080
            initialDelaySeconds = 30
        }
        configMapRef = "${name}-config"
        serviceAccountName = name
    })
    _service = svc.build_service(svc.ServiceSpec {
        name = name; namespace = namespace; port = 8080
    })
    _configmap = cm.build_configmap(cm.ConfigMapSpec {
        name = "${name}-config"
        namespace = namespace
        data = { "application.yaml" = "server.port: 8080" }
    })
    _sa = sa.build_service_account(sa.ServiceAccountSpec {
        name = name; namespace = namespace
    })
    _leader = leader.build_component_leader(name, namespace)
    leaders = [_leader]
    manifests = [_deployment, _service, _configmap, _sa]
```

```kcl
# AFTER (template pattern — ~50 lines):
import framework.templates.webapp as webapp
import framework.builders.deployment as deploy

schema MyAppModule(webapp.WebAppModule):
    port = 8080
    replicas = 2
    serviceType = "ClusterIP"
    configData = { "application.yaml" = "server.port: 8080" }
    env = [
        { name = "SPRING_PROFILE", value = "dev" }
        { name = "DB_HOST", value = "postgres.data.svc" }
    ]
    resources = deploy.ResourceSpec {
        cpuRequest = "250m"
        memoryRequest = "512Mi"
        cpuLimit = "1"
        memoryLimit = "2Gi"
    }
    livenessProbe = deploy.ProbeSpec {
        probeType = "http"
        path = "/health"
        port = 8080
        initialDelaySeconds = 30
    }
```

### Step 3: Compare Outputs

```bash
# Render old module
cd projects/video_streaming/pre_releases/manifests/dev/factory
kcl run render.k -D output=yaml > /tmp/old_output.yaml

# Render new module (after replacing in stack)
kcl run render.k -D output=yaml > /tmp/new_output.yaml

# Diff the outputs
diff /tmp/old_output.yaml /tmp/new_output.yaml
```

### Step 4: Replace Module Reference in Stack

Update the stack definition to use the new module's `.instance`:

```kcl
# stacks/stack_def.k
import modules.my_app as app  # Updated module

_stack = stack_model.Stack {
    components = [app.my_app_module.instance]  # Same .instance pattern
}
```

### Step 5: Remove Old Module File

Once validated with `kubeconform` and `diff`, remove the old raw module definition file.

## Priority Order

| Module | Type | Complexity | Notes |
|---|---|---|---|
| `kafka_video_consumer_mongodb_python` | APPLICATION | Medium | Env vars, MongoDB deps → `WebAppModule` |
| `mongodb_single_instance` | INFRASTRUCTURE | Low | Basic DB → `SingleDatabaseModule` or `MongoDBCommunityModule` |
| `kafka_strimzi` | CRD | Medium | Requires `KafkaClusterModule` |

---

## Validation Checklist

For each migrated module:

- [ ] New template-based module compiles: `kcl run module_def.k`
- [ ] YAML output matches original: `diff` old vs new
- [ ] Stack compiles with new module: `koncept validate`
- [ ] Full render works: `koncept render argocd`
- [ ] kubeconform validates output: `kubeconform -summary`
- [ ] Old module file removed
- [ ] Stack reference updated to new `.instance`
